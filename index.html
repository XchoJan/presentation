Marketolog — презентация проекта
Marketolog · smmpromote.co

AI-маркетолог, который сам ведёт Instagram и Facebook вашего бизнеса

B2B SaaS-платформа: искусственный интеллект генерирует посты с изображениями под нишу и бренд клиента, администратор модерирует, планировщик публикует через Meta Graph API. Без публичной регистрации — аккаунты выдаются вручную.

02 Продукт

SMM-отдел в одной подписке

Малому бизнесу нужен контент каждый день, но нанимать маркетолога дорого. Marketolog закрывает весь цикл: придумать → написать → нарисовать → согласовать → опубликовать — от клиента требуется только описать свой бизнес и нажать «Сгенерировать».

03 Бизнес-модель

Управляемые аккаунты, месячная подписка

Публичной регистрации нет — это осознанное продуктовое решение. Администратор сам создаёт клиента, выдаёт логин и пароль и включает доступ после оплаты. Продажа и онбординг остаются в руках владельца сервиса.

Админ создаёт аккаунт

Email или username, временный пароль, роль. Клиент получает доступ вручную — ни signup-страницы, ни self-service сброса пароля.

Оплата = доступ

Кнопка «Отметить оплату» продлевает подписку ровно на месяц (subscriptionExpiresAt). Не оплачено или истекло — кабинет закрыт.

Дневной лимит

6 генераций в календарный день по таймзоне клиента, сброс в его полночь. Админ может обнулить лимит вручную в один клик.

04 Как это работает

От логина до опубликованного поста — пять шагов

01

Вход и профиль бизнеса

Клиент входит по выданным данным и заполняет карточку: название, ниша, описание, сайт, фирменный цвет. Эти поля — сырьё для всех промптов ИИ.

02

Подключение соцсетей

Facebook Page и/или Instagram Business — по access-токену и ID страницы. Встроенные пошаговые гайды со скриншотами и проверка подключения (/social/verify) прямо в интерфейсе.

03

Генерация контента

Одна кнопка — и ИИ выдаёт 2 готовых поста: заголовок, продающий текст, хэштеги и изображение 1:1, привязанное к теме поста и бренд-цвету. Система помнит недавние посты и не повторяется.

04

Модерация

Посты попадают в очередь со статусом «На согласовании». Администратор одобряет, отклоняет или ставит расписание публикации.

05

Публикация

Сразу или по расписанию: фоновый воркер каждую минуту проверяет запланированные посты и отправляет их в Meta Graph API. В посте сохраняется externalId из соцсети.

05 AI-конвейер

Не «текст по шаблону», а контент-стратегия

Генерация — двухступенчатая: GPT (по умолчанию gpt-4o-mini) пишет тексты, gpt-image-1 / DALL-E рисует изображение по промпту, собранному из темы поста, ниши и фирменного цвета. Файлы сохраняются в public/generated/ — у постов стабильные URL.

Тексты с ротацией форматов

10 редакционных форматов — обучающий совет, разрушение мифа, закулисье, чек-лист, инсайт индустрии, ответ на возражение, кейс, мягкое промо, сторителлинг, полезный факт. Плюс контекст последних постов, чтобы лента не превращалась в копипасту.

Изображения по теме поста

Промпт собирается детерминированно: компания, индустрия, тема поста («must be visually obvious»), сцена, бренд-цвет в освещении. Никаких случайных стоков — без текста, логотипов и водяных знаков.

Три языка контента

Русский, English, Հայերեն — язык выбирается на бизнесе и прокидывается в промпт. Хэштеги нормализуются автоматически.

Мок-режим без ключа

Нет OPENAI_API_KEY — работает встроенный генератор с плейсхолдерами picsum. Проект полностью демонстрируется без единого внешнего ключа.

06 Жизненный цикл поста

Каждый пост проходит контроль качества

Ничего не улетает в соцсети без одобрения: статусная машина из шести состояний отделяет генерацию от публикации, а модерацию — от планирования.

DRAFT · черновик → PENDING_APPROVAL · на согласовании → APPROVED · одобрен → SCHEDULED · запланирован → PUBLISHED · опубликован ⤷ REJECTED · отклонён

Гигиена очереди

В очереди клиента до 50 постов; опубликованные старше 30 дней удаляются автоматически (настраивается через POST_RETENTION_DAYS).

Следы публикации

После публикации сохраняются publishedAt и externalId, который вернул Meta Graph API, — путь поста прослеживается до соцсети.

Очистка одним кликом

Клиент может очистить очередь черновиков (POST /api/posts/clear), не трогая запланированное и опубликованное.

07 Публикация и планировщик

Meta Graph API + фоновый воркер

Прямая публикация

Graph API v21.0: пост уходит на Facebook-страницу и/или в Instagram Business — режим выбирается на публикации. Для Instagram — двухфазный контейнерный flow, page-токен резолвится автоматически, ошибки Meta переводятся в понятные сообщения.

Воркер расписания

Отдельный процесс (npm run worker, интервал 60 секунд) находит посты со статусом SCHEDULED, чьё время пришло, публикует их и попутно чистит устаревшие. Запускается и вручную — из админки или через POST /api/scheduler/run.

Демо-симуляция

Без реального токена Meta публикация симулируется: статус меняется на PUBLISHED, проставляется фиктивный externalId. Весь flow можно показать заказчику без доступа к соцсетям.

Проверка перед стартом

Публикация не начнётся без готовой соцсети: система проверяет токен + ID страницы для каждого выбранного канала и точно объясняет, чего не хватает.

08 Интерфейс · клиент

Кабинет из трёх экранов

Тёмно-синяя тема, минимум действий: клиенту не нужно разбираться в маркетинге — только смотреть и подтверждать.

Дашборд

Сводка: статистика постов по статусам, состояние подписки, остаток дневного лимита с обратным отсчётом до сброса, быстрые действия.

Очередь контента

Карточки постов с изображением, текстом, хэштегами и цветным статус-бейджем. Кнопка генерации, скачивание картинки, публикация, очистка очереди.

Бизнес-профиль

Карточка бизнеса + подключение соцсетей: поля токенов, проверка соединения и модальные гайды «как получить токен» для Facebook и Instagram.

09 Интерфейс · админ

Админ-панель — пульт управления бизнесом

Пользователи

Создание и редактирование аккаунтов, активация/блокировка, отметка оплаты (+1 месяц подписки), сброс дневного лимита. Статус подписки виден по цветному бейджу.

Бизнесы

Все бизнес-профили клиентов в одном месте: создание, привязка к пользователю, редактирование ниши и брендинга.

Модерация контента

Лента всех постов платформы: одобрить, отклонить, назначить время публикации или опубликовать немедленно — с фильтрами по статусу.

Планировщик вручную

Кнопка «Запустить планировщик» — прогнать цикл публикации без ожидания воркера. Удобно для демо и отладки.

12 Инфраструктура

Продакшен на Docker: три контейнера и прокси

Проект уже развёрнут на smmpromote.co. Docker Compose поднимает приложение, воркер и PostgreSQL; Traefik терминирует HTTPS с автоматическими сертификатами Let's Encrypt.

  • app — Next.js-сборка, prisma db push при старте, порт 3000 за Traefik
  • worker — тот же образ, команда npm run worker, интервал из WORKER_INTERVAL_MS
  • db — PostgreSQL в приватной docker-сети
  • Traefik — маршрутизация smmpromote.co + www, TLS-резолвер letsencrypt
  • deploy.sh — доставка на сервер одним скриптом
  • Секреты через окружение: JWT_SECRET обязателен, OPENAI_API_KEY опционален
# локальный запуск
$ npm install
$ npm run db:push   # схема БД
$ npm run db:seed   # демо-данные
$ npm run dev       # localhost:3000
$ npm run worker    # планировщик

# продакшен
$ docker compose -f docker-compose.infra.yml up -d
$ docker compose -f docker-compose.app.yml up -d --build
